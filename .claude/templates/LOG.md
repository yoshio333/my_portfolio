# 作業ログ

セッションごとに追記していく。**新しいものが上。**
作業が一区切りついたら**その場で**書く（セッション末にまとめて書かない）。

書き方のポイント：
- 「何をしたか」より「**なぜそうしたか**」を書く
- 症状 → 原因 → 修正 の3点セットが理想
- コードの変更はファイルに残る。ログに残すべきは**判断の理由**
